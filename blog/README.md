## 上线规则
### 上线不规范导致的问题记录

- 问题
  - 部分用户出现信息更新失败的情况
- 原因
  - 一个迭代中的新功能部署正式，上线后需要跑运维脚本，当时在忙其他工作，认为不会对其他功能造成影响，手上工作忙完后再执行脚本，在上线后到跑运维脚本之间，延期会触发某个字段更新操作，由于初始化数据有validation验证，导致更新失败
- 解决措施
  - 有问题的用户，手动执行更新操作
- 后续可改进的
  - 功能上线后，应及时跑运维脚本
  - 有数据运维的功能上线，尽量选用户使用少的时间，防止出现意外问题，将损失降到最低
  - 对于初始化数据的 Validations和Callbacks，综合全面考虑可能带来的影响
